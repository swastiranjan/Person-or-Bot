Person or Bot Detection using Machine Learning

This project implements a machine learning model to classify whether typing behavior belongs to a human or a bot. The system analyzes typing behavior features from a dataset and uses a Random Forest classifier to predict whether the input pattern is generated by a person or an automated bot.

The goal of this project is to demonstrate how Artificial Intelligence and Machine Learning can be used for behavioral analysis and bot detection, which is useful in cybersecurity, fraud detection, and automated system prevention.

Project Objectives

Load and analyze a typing behavior dataset

Train a machine learning model to classify human vs bot activity

Evaluate model performance using accuracy and cross-validation

Analyze prediction results using a confusion matrix

Technologies Used

Python

Pandas (Data handling)

Scikit-learn (Machine Learning)

Random Forest Classifier

Jupyter Notebook

Dataset

The dataset person_or_bot.csv contains features describing typing behavior.
These features are used to determine whether the activity is performed by a human or an automated bot.

Typical dataset structure:

Feature 1	Feature 2	Feature 3	...	Label
value	value	value	...	person/bot

Features: Typing behavior metrics

Label: Classification target (Person or Bot)

Project Workflow
1. Import Required Libraries

Libraries such as pandas and scikit-learn are imported to handle data processing and model training.

2. Load Dataset

The dataset is loaded using pandas and inspected to verify its structure.

3. Feature and Label Separation

The dataset is divided into:

X → Input features

y → Target label (person or bot)

4. Train-Test Split

The dataset is divided into:

Training set (65%)

Testing set (35%)

This ensures the model is evaluated on unseen data.

5. Model Training

A Random Forest Classifier is used because it:

Handles complex datasets well

Reduces overfitting

Provides strong classification performance

6. Prediction

The trained model predicts labels for the test dataset.

7. Model Evaluation

The model is evaluated using:

Accuracy Score

Measures the percentage of correct predictions.

Confusion Matrix

Shows the number of correct and incorrect classifications.

Example structure:

	Predicted Person	Predicted Bot
Actual Person	TP	FN
Actual Bot	FP	TN
8. Cross Validation

5-fold cross validation is used to verify the model’s reliability across different dataset splits.

This provides a more robust estimate of model performance.
